# 텍스트 파일 vs 이진 파일 비교

## 1. 텍스트 파일 (CSV 형식)

**정의**
- 인간이 읽을 수 있는 문자 형태로 저장되는 파일

**특성**
- 파일 크기: 이진 파일보다 크다
- 읽기 속도: 상대적으로 느리다 (파싱 과정 필요)
- 호환성: 다양한 프로그램/언어에서 쉽게 읽을 수 있다

**장점**
- 사람이 직접 읽고 수정할 수 있다
- 버전 관리(Git 등)에 적합하다
- 데이터 교환에 표준화되어 있다
- 다양한 운영체제 간 호환성이 좋다

**단점**
- 파일 크기가 크다
- 처리 속도가 상대적으로 느리다
- 타입 정보가 명시적이지 않다
- 불필요한 문자 변환 과정이 필요하다

---

## 2. 이진 파일 (Binary 형식)

**정의**
- 0과 1의 바이너리 형태로 저장되는 파일

**특성**
- 파일 크기: 텍스트 파일보다 작다
- 읽기 속도: 상대적으로 빠르다 (직접 메모리에 로드)
- 호환성: 같은 언어/라이브러리에서만 읽을 수 있다

**장점**
- 파일 크기가 작다
- 처리 속도가 빠르다
- 객체 구조가 그대로 보존된다 (직렬화)
- 메모리 효율성이 높다

**단점**
- 직접 읽을 수 없다 (디코딩 필요)
- 버전 관리에 적합하지 않다
- 프로그램 간 호환성이 낮다
- 언어나 플랫폼이 다르면 호환되지 않을 수 있다

---

## 3. 사용 시나리오

### 텍스트 파일 추천
- 데이터 교환이 필요한 경우
- 버전 관리(Git 등)가 필요한 경우
- 사람의 검토와 수정이 필요한 경우
- 서로 다른 프로그램 간의 상호 운용성이 중요한 경우
- 예시: CSV, JSON, XML, 설정 파일 등

### 이진 파일 추천
- 대용량 데이터를 자주 읽는 경우
- 빠른 처리 속도가 필요한 경우
- 복잡한 객체 구조를 보존해야 하는 경우
- 메모리 효율성이 중요한 경우
- 예시: 이미지, 동영상, 오디오, 직렬화된 객체 등

---

## 4. 성능 비교

| 항목 | 텍스트 파일 | 이진 파일 |
|------|-----------|---------|
| 파일 크기 | 크다 | 작다 |
| 읽기 속도 | 느리다 | 빠르다 |
| 쓰기 속도 | 느리다 | 빠르다 |
| 호환성 | 높다 | 낮다 |
| 수정 용이성 | 높다 | 낮다 |
| 디버깅 용이성 | 높다 | 낮다 |

---

## 5. 결론

**텍스트 파일 선택 기준**
- 데이터 교환이 중요한 프로젝트
- 코드 검토와 버전 관리가 필요한 경우
- 사람이 직접 데이터를 확인해야 하는 경우

**이진 파일 선택 기준**
- 성능과 속도가 중요한 대규모 데이터 처리
- 메모리 효율성이 중요한 경우
- 같은 프로그램 내에서만 사용하는 데이터 저장
